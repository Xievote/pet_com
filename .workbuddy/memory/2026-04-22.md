# 2026-04-22 工作日志

## 修复「非法请求」问题
- 项目：tp5（ThinkPHP5 宠物网站 demo）
- 问题：`localhost:8080/index/pet/login.html` 报「非法请求:index/pet/login」
- 真正根因：`application/index/view/pet/index.html` 第72行表单 action 写的是 `url('index/pet/save')` 全路径，而 route.php 里已注册了 `pet/save` 路由，TP5 检测到命名路由已存在时禁止通过完整路径绕路访问，抛出 invalid request。
- 修复：将 `url('index/pet/save')` 改为 `url('pet/save')`，与 route.php 中的路由定义保持一致。
- 经验：凡是在 route.php 中注册了路由别名的，模板里的 url() 必须用别名路径，不能用 `模块/控制器/方法` 全路径。
